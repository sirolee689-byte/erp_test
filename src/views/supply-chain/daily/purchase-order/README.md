# 采购订单模块

## 已完成功能

- 采购单列表：默认显示 `del=0` 且 **已审核**（`pass=1`）的正常单据；筛选区两行布局（对齐入库单体验）：
  - 第一行：**供应商**（可搜索下拉，按编码筛选）+ **采购类型**
  - 第二行：**查询内容**（单号/PI/供应商/备注）+ 查询/重置 + 回收站开关 + 显示未审核开关（竖线间隔）
  - 打开「显示未审核」只看 `pass=0`；打开「回收站」只看 `del=1` 的软删除单
- 列表分页默认 `10` 条/页，支持切换 `10/20/50/100`。
- 列表按 `addtime` 最新优先排序（同时间再按 `id` 倒序）。
- 列表字段顺序改为：操作、状态、采购类型、采购订单号、采购订单日期、采购订单数据、关联单号、币别、供应商/外协商、是否含税、装货港、卸货港、付款条件、备注、录入人（`utruename`）。
- 「采购订单数据」列宽 `380px`，分 4 行展示：供应商+小数位、总项数/总数量/未审数、含税/不含税/税点总价、额外费用；未审数大于 0 红色显示，等于 0 蓝色显示。
- 页面顶部提供「管理采购订单 / 采购订单添加」切换（对齐外协订单）；添加/编辑页右上角固定提供「返回列表 / 保存」，不再在表单底部显示立即提交和重置；添加页分三个页签：订单基础资料、采购订单明细、额外费用清单。
- 订单基础资料页签采用 8 行业务表单：
  - 第一行：采购单号 + 单号类型（`ZY` / `PO` / 当前年份）；
  - 第二行：采购日期 + 采购类型（订单采购 / 请购采购 / 其他采购）；
  - 第三行：关联单号输入框 + 选择 PI + 清空，支持手填和多 PI 选择；
  - 第四至八行依次为供应商、装货港/卸货港、是否含税/币别/小数点配置、付款条件、备注。
- 列表使用视口横向滚动条（`ErpTableViewportHScroll`），表格不设固定高度，避免列表区域出现纵向双滚动条；分页在表格上下各一组。
- 操作列按钮样式对齐外协订单（小号 plain 按钮）；已去掉结案/反结案；编辑与删除仅在「显示未审核」视图出现；反审仅在已审核列表出现。
- 单号规则：支持 `ZY`、`PO`、当前年份三类；`ZY` 默认订单采购可切请购采购，`PO` 锁定请购采购，年份号默认请购采购但可切其他类型。年份号使用“年份 + 5 位序号”，例如 `202600166`。
- 关联单号：订单采购和请购采购必须填写；支持手动文本，也支持选择 PI。PI 弹窗默认 `10` 条/页，并按销售单 `addtime` 最新优先排列；第一列为「操作」并按行提供「选择」按钮；订单采购/其他采购点选后立即带回单个 PI，请购采购允许连续选择多个 PI，选中后按钮变为「已选择」并换色，再次点击可取消，顶部「查询」右侧显示「保存已选数据」，确认后用逗号保存多个 PI。
- 明细：从 `UB_ERP_Bom_000` 选择物料，保存到 `UB_ERP_Buy_order_list`，`kcak02/systemcode` 使用 BOM 的 `systemcode`。
- 采购订单明细 Tab 提供 `删除选定明细`、`删除全部明细`、`批量添加` 三个按钮；删除只影响当前页面，点击右上角保存后才落库。
- 采购订单明细表格列顺序（添加/编辑页统一）：选择（橙色标记删行）→ 序号 → 操作（查看原资料）→ 材料编码 → 材料名称/送货名（上下分层）→ 材料规格 → 颜色 → 采购单位 → 数量 → 单价 → 单价(含税) → 金额 → 金额(含税) → 税点 → PO/PI → 交货日期 → 备注；数量、含税单价、税点输入框无 `+/-` 按钮；PO/PI 只读展示（PO 来自所选 PI 对应销售单、PI 来自关联单号），保存时明细 `Reference` 仍写 PI 号。
- `删除选定明细`：先在「选择」列点橙色「删除」标记行，再批量删除；标记状态 `_lineMarked` 不入库。
- `查看原资料`：订单采购且有关联 PI 时，新页打开 PI-BOM（`/inventory/basic/pi-bom-data-window`）；缺 PI 或款号时提示。
- `订单采购` 支持从当前单个 PI 批量添加采购物料：新窗口读取 `UB_ERP_Bom_pi_cost.sid = PI号` 且 `kcaa12=1` 的采购物料，按物料编码聚合并按 `px` 升序显示；选择后带回建议采购数量、供应名、采购单位、报价单价、已采购数量和已入库数量；库存数量/扣数后库存数列先保留为 0，不依赖不存在的旧库存汇总表。
- 批量添加新页默认 `10` 条/页，支持 `10/20/50/100`；接口 `GET /api/buy-order/batch-add-lines` 返回 `total` + `list`，翻页查询；跨页已选物料保留在页面内存中。
- `订单采购` 批量添加接口（`GET /api/buy-order/batch-add-lines`）已修复：`UB_ERP_Bom_pi_cost` 与报价/采购明细表共享 `kcaa07` 等数值字段名，聚合时须保持 decimal 类型，避免与 `OUTER APPLY` 子查询发生 nvarchar→numeric 转换错误。
- 额外费用：费用必须来自已审核、未删除且分类为 `FEE` 的 BOM 资料，保存到 `UB_ERP_Buy_order_money`。
- 审核/反审/删除：后端按 `pass`、`closed`、`del` 和入库单引用关系做强校验；反审必须填写原因。列表页已隐藏结案/反结案按钮（后端能力仍保留）。
- 查看/打印：支持只读详情和打印预览；价格字段按 `price` 权限显示或隐藏。
- 入库联动：采购入库来源改为读取采购单真实字段 `kcaj01/kcaj05`，入库明细 `kcao02` 使用采购明细 `kcak02`。

## 数据库口径

- 主表：`UB_ERP_Buy_order`
- 明细：`UB_ERP_Buy_order_list`
- 额外费用：`UB_ERP_Buy_order_money`
- 反审原因：`UB_ERP_Buy_order_sp`
- 订单 BOM 快照：`UB_ERP_Bom_buy_order`、`UB_ERP_Bom_buy_order_list`
- 入库引用：`UB_ERP_Stocks_Storage.kcan04 = UB_ERP_Buy_order.kcaj01`

## 已知问题 / 下一步

- 第一版不做批量审核、审核驳回状态、真实 Excel 导出、超 30000 RMB 邮件、超订量申请复核。
- 页面先满足业务闭环，后续可以继续按外协订单的细节体验做密度和批量选材优化。
- 采购批量添加窗口只支持订单采购的单 PI；请购采购的多 PI 批量添加留到后续版本。批量带回的可采购数量是建议值，保存端仍只强校验采购数量大于 0。
- 若批量添加仍报类型转换错误，先确认 3001 后端已重启并重新登录，再检查 PI 运算数据是否完整。
