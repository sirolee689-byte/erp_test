# 采购订单模块

## 已完成功能

- 采购单列表：默认显示 `del=0` 且 **已审核**（`pass=1`）的正常单据；筛选区两行布局（对齐入库单体验）：
  - 第一行：**供应商**（可搜索下拉，按编码筛选）+ **采购类型**
  - 第二行：**查询内容**（单号/PI/供应商/备注）+ 查询/重置 + 回收站开关 + 显示未审核开关（竖线间隔）
  - 打开「显示未审核」只看 `pass=0`；打开「回收站」只看 `del=1` 的软删除单
- 列表分页默认 `10` 条/页，支持切换 `10/20/50/100`。
- 列表按 `addtime` 最新优先排序（同时间再按 `id` 倒序）。
- 列表字段顺序改为：操作、状态、采购类型、采购订单号、采购订单日期、采购订单数据、关联单号、币别、供应商/外协商、是否含税、装货港、卸货港、付款条件、备注、录入人（`utruename`）。
- 「采购订单数据」列宽 `380px`，分 4 行展示：供应商+小数位、总项数/总数量/未审数、含税/不含税/税点总价、额外费用；未审数大于 0 红色显示，等于 0 蓝色显示。
- 页面顶部提供「管理采购订单 / 采购订单添加」切换（对齐外协订单）；**添加页**右上角为「重置 / 保存」，**编辑页**为「返回列表 / 保存」；添加页点「重置」会清空基础资料、采购明细与额外费用（有明细或费用时先确认）；回列表用顶部「管理采购订单」；添加页分三个页签：订单基础资料、采购订单明细、额外费用清单。
- 订单基础资料页签采用 8 行业务表单：
  - 第一行：采购单号 + 单号类型（`ZY` / `PO` / 当前年份）；
  - 第二行：采购日期 + 采购类型（订单采购 / 请购采购 / 其他采购）；
  - 第三行：关联单号输入框 + 选择 PI + 清空，支持手填和多 PI 选择；
  - 第四至八行依次为供应商、装货港/卸货港、是否含税/币别/小数点配置、付款条件、备注。
- 列表使用视口横向滚动条（`ErpTableViewportHScroll`），表格不设固定高度，避免列表区域出现纵向双滚动条；分页在表格上下各一组。
- 操作列按钮样式对齐外协订单（小号 plain 按钮）；已去掉结案/反结案；编辑与删除仅在「显示未审核」视图出现；反审仅在已审核列表出现。
- 单号规则：支持 `ZY`、`PO`、当前年份三类；`ZY` 默认订单采购可切请购采购，`PO` 锁定请购采购，年份号默认请购采购但可切其他类型。年份号使用“年份 + 5 位序号”，例如 `202600166`。
- 关联单号：订单采购和请购采购必须填写；支持手动文本，也支持选择 PI。PI 弹窗默认 `10` 条/页，并按销售单 `addtime` 最新优先排列；第一列为「操作」并按行提供「选择」按钮；订单采购/其他采购点选后立即带回单个 PI，请购采购允许连续选择多个 PI，选中后按钮变为「已选择」并换色，再次点击可取消，顶部「查询」右侧显示「保存已选数据」，确认后用逗号保存多个 PI。
- 明细：从 `UB_ERP_Bom_000` 选择物料，保存到 `UB_ERP_Buy_order_list`，`kcak02/systemcode` 使用 BOM 的 `systemcode`。
- 采购订单明细 Tab 提供 `删除选定明细`、`删除全部明细`、`批量添加` 三个按钮；删除只影响当前页面，点击右上角保存后才落库。
- 采购订单明细表格列顺序（添加/编辑页统一）：选择（橙色标记删行）→ 序号 → 操作（查看原资料）→ 材料编码 → 材料名称/送货名（上下分层）→ 材料规格 → 颜色 → 采购单位 → 数量 → 单价 → 单价(含税) → 金额 → 金额(含税) → 税点 → PO/PI → 交货日期 → 备注；数量、含税单价、税点输入框无 `+/-` 按钮；PO/PI 只读展示（PO 来自所选 PI 对应销售单、PI 来自关联单号），保存时明细 `Reference` 仍写 PI 号。
- `删除选定明细`：先在「选择」列点橙色「删除」标记行，再批量删除；标记状态 `_lineMarked` 不入库。
- `查看原资料`：订单采购且有关联 PI 时，新页打开 PI-BOM（`/inventory/basic/pi-bom-data-window`）；缺 PI 或款号时提示。
- `订单采购` 支持从当前单个 PI 批量添加采购物料：新窗口读取 `UB_ERP_Bom_pi_cost.sid = PI号` 且 `kcaa12=1` 的采购物料，按物料编码聚合并按 `px` 升序显示；选择后带回建议采购数量、供应名、采购单位、报价单价、已采购数量和已入库数量；库存数量/扣数后库存数列先保留为 0，不依赖不存在的旧库存汇总表。
- `请购采购` 与 `其他采购` 共用同一个批量添加弹窗：两者都须先选供应商；新窗口默认 **10 条/页**；单 PI、空关联号或无逗号时从 `UB_ERP_Bom_000` 全量选材（`pass=1`、排除裁片），多 PI（逗号分隔）时改读 `UB_ERP_Bom_Sales_list` 并显示 SI 列；多 PI 下相同材料编码（`kcaa01`，如 `LA-0368/BO`）在接口分页前汇总为一行，SI 列显示去重后的来源 PI；顶部提供分类下拉（`UB_ERP_Bom_code` 前缀过滤）、关键词搜索、全选（一次最多 300 条）；查询条件只按材料编码 `kcaa01` 搜索；弹窗列表不查报价，单价列显示「待带出」，点击「保存已选数据」时再按已选物料批量读取 `UB_ERP_Buy_offer_list` 最新报价并回填；不带采购量/已采购/库存列；带回明细数量为 **0**，须手工录入；有报价则自动带单价/税点。
- 批量添加物料弹窗在规格列右侧展示完整物料扩展字段：开票名、单位、分类、客户/工厂款号、库存上下限、颜色、采购/外协/自产、海关信息、包装重量、采购/报价单位与转换、损耗、币别、备注、审核状态、产地、价格和客户供应信息；分类从 `UB_ERP_Stocks_material` 转名称，报价/采购币别从 `UB_ERP_Finance_currency` 转名称，查不到时保留原编码显示。
- 批量添加新页默认 `10` 条/页，支持 `10/20/50/100`；接口 `GET /api/buy-order/batch-add-lines` 返回 `total` + `list`，翻页查询；跨页已选物料保留在页面内存中。
- `订单采购` 批量添加接口（`GET /api/buy-order/batch-add-lines`）已修复：`UB_ERP_Bom_pi_cost` 与报价/采购明细表共享 `kcaa07` 等数值字段名，聚合时须保持 decimal 类型；搜索 `kcaa01~kcaa35` 时统一先转文本再 `LIKE`，避免数字字段触发 nvarchar→numeric 转换错误。
- `订单采购` 批量添加性能边界：接口先按 PI 聚合并完成 `ROW_NUMBER` 分页，再只对当前页物料补充供应名、最新报价、已采购数量、已入库数量；供应名 `gkcaa02` 必须按当前页物料批量查最近采购明细，不要恢复为每行 `TOP 1` 扫历史采购明细；也不要把报价/入库等 `OUTER APPLY` 提前到分页前，否则大 PI 会先补全全部物料再显示 10 条，加载会明显变慢。
- `请购采购`/`其他采购` 批量添加性能边界：最慢的关联是最新报价 `UB_ERP_Buy_offer_list JOIN UB_ERP_Buy_offer`，因此列表加载、搜索、翻页阶段不关联报价表；多 PI 时的第二慢点是 `UB_ERP_Bom_Sales_list` 按 `kcaa01` 分组并用 `STUFF + FOR XML PATH` 拼接来源 PI。
- 额外费用：费用必须来自已审核、未删除且分类为 `FEE` 的 BOM 资料，保存到 `UB_ERP_Buy_order_money`；添加/编辑表单中额外费用表仅显示费用编码、费用名称、金额、备注；不展示规格与税点列（保存时规格仍从 BOM 带出、税点按 0）；金额列为普通输入框，默认空白、无加减按钮，0 显示为空；落库时 `systemcode` 与主表 `UB_ERP_Buy_order.systemcode` 一致，并写入当前操作员 `uid`/`uname`/`utruename` 与保存时间 `addtime`（整批重写时按本次保存刷新）。
- 审核/反审/删除：后端按 `pass`、`closed`、`del` 和入库单引用关系做强校验；审核、删除、恢复、彻底删除在前端先弹确认框，反审必须填写原因。列表页已隐藏结案/反结案按钮（后端能力仍保留）。
- 查看/打印：支持只读详情和打印预览；价格字段按 `price` 权限显示或隐藏。
- 入库联动：采购入库来源改为读取采购单真实字段 `kcaj01/kcaj05`，入库明细 `kcao02` 使用采购明细 `kcak02`。

## 数据库口径

- 主表：`UB_ERP_Buy_order`
- 明细：`UB_ERP_Buy_order_list`
- 额外费用：`UB_ERP_Buy_order_money`
- 反审原因：`UB_ERP_Buy_order_sp`；反审原因写入真实列 `oid/content/kcaj01` 等，其中 `oid` 关联 `UB_ERP_Bom_buy_order.id`，不使用旧写法 `xsaj01`。
- 订单 BOM 快照：`UB_ERP_Bom_buy_order`、`UB_ERP_Bom_buy_order_list`
- 入库引用：`UB_ERP_Stocks_Storage.kcan04 = UB_ERP_Buy_order.kcaj01`

## 已知问题 / 下一步

- 第一版不做批量审核、审核驳回状态、真实 Excel 导出、超 30000 RMB 邮件、超订量申请复核。
- 页面先满足业务闭环，后续可以继续按外协订单的细节体验做密度和批量选材优化。
- 采购批量添加窗口：订单采购仅支持单 PI；请购采购和其他采购都支持 BOM 全量或多 PI 销售清单选材（其他采购可不填关联号直接全库选材）。批量带回的可采购数量（订单采购）是建议值；请购/其他采购带回数量默认为 0。保存端仍只强校验采购数量大于 0。
- 若批量添加仍报类型转换错误，先确认 3001 后端已重启并重新登录，再检查 PI 运算数据是否完整。
- **展开明细/列表汇总数字安全读取（2026-06-18）**：`expand-detail` 与列表聚合 SQL 已改用 `buyOrderSqlSafe.safeDecimalExpr`，避免旧表 nvarchar 数量/单价字段触发 `nvarchar→numeric` 转换错误。
- **展开明细 kcak06 文本读取（2026-06-22）**：明细表 `kcak06`（上级材料编码）物理类型为 `numeric`，不能再写 `ISNULL(kcak06, N'')`；已改用 `nvarcharTextExpr` 先转文本再兜底空串。
- **BOM 快照 GUID（2026-06-18）**：保存时 `UB_ERP_Bom_buy_order.[GUID]` 与 `[systemcode]` 同值写入，均取自 `UB_ERP_Bom_000.GUID`（无 GUID 时用 systemcode 兜底）。

## 最近体验规则

- 采购订单明细数量按两位小数保存和展示；数量为 `0` 时在明细输入框和展开明细里显示为空白，方便用户直接录入。
- 点击保存时，订单基础资料的必填项未填完整，会先提示并自动切回“订单基础资料”页签。
- 保存明细会写入 `UB_ERP_Buy_order_list.uid/uname/utruename/addtime`；保存订单 BOM 快照会写入 `UB_ERP_Bom_buy_order` 的 `GUID`/`systemcode`（同值）、扩展物料字段、审核字段和录入人字段。

## 管理页展开明细
- 管理采购订单主页面支持点击行展开明细，展开数据来自 `GET /api/buy-order/:id/expand-detail`。
- 展开表同时显示采购明细和额外费用；采购明细展示材料、数量、入库相关、价格字段、PO/PI、交货日期、客户订单号和备注。
- 展开明细价格列中，不含税单价列显示为“单价”，不含税金额列显示为“金额”；含税列仍显示“单价（含税）”“金额（含税）”。
- 入库相关按采购单号和物料批量聚合展示，兼容 `kcaa01` 材料编码和 `kcao02 = kcak02/systemcode` 两种匹配口径；退货只做红色历史提示，不在采购单里反写库存。
- 展开明细的“查看”按钮优先打开 PI-BOM 查看页面；如果当前行缺少 PI 或无法解析销售订单，会提示用户而不是跳到错误页面。
## 2026-06-22 删除体验修复
- 点击采购单“删除”时，后端不再固定写旧字段 `delid`。
- 删除时先读取 `UB_ERP_Buy_order` 当前真实字段，`delname`、`deltruename`、`deltime` 等字段存在才写；不存在就跳过。
- 用户看到的变化：未审核、未入库关联的采购单可以正常删除到回收站，不会再弹 `Invalid column name 'delid'`。
