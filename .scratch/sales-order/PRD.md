# 销售订单模块 PRD

**Status:** `ready-for-agent`  
**来源：** `CONTEXT.md` 第七节「销售订单」（已定稿域模型）  
**域词汇：** PI 号、系统单号、销售订单明细、主 BOM、PI 销售 BOM（PI BOM）、同步 BOM、一键运算、物料单、订货数量、运算状态  

---

## Problem Statement

供应链日常业务需要按 **PI（Proforma Invoice）** 管理销售订单：录入客户、币别、多款成品及订货数量，并在 **每张 PI 内独立维护 BOM**（与库存主 BOM 分离、可按单改用量），再经 **一键运算** 生成全系统统一的 **物料单** 供仓库、采购、出入库订料。

当前系统仅有销售订单菜单占位页，无后端接口与完整前端；域规则已在 `CONTEXT.md` 定稿，但缺少可执行的 PRD 以指导端到端实现。

---

## Solution

交付一套 **主从业务单据** 模块（对齐采购报价金标准），包含：

1. **销售订单主从**：`UB_ERP_Sales_order` + `UB_ERP_Sales_order_list`，支持审核/软删/回收站/彻底删除。
2. **PI 专用 BOM**：`UB_ERP_Bom_Sales` + `UB_ERP_Bom_Sales_list`，按款与订单明细一一对应；保存时 **PI BOM 对齐**（删款物理删头+子件），**禁止** 整 PI 先删后插；**主 BOM 门禁** 控制何时可从主 BOM 写入。
3. **同步 BOM**（按行、手动）与 **一键运算**（读 PI BOM 写物料单），主表 **运算状态**（已运算/未运算）驱动下游是否可订料。
4. **物料单**：`UB_ERP_Bom_pi_cost`（明细）+ `UB_ERP_Bom_pi_consumption`（按子件编码+备注汇总）；运算 **不乘订货数量**，展示/订料时 **用量 × 订货数量**。

用户在同一 PI 内可改 BOM 而不被保存偷偷覆盖；只有显式 **同步 BOM** 或删行后同码再加等例外才从主 BOM 拉取。

## User Stories

### 列表与检索

1. 作为跟单员，我想在列表页分页查看销售订单（系统单号、PI 号、客户、日期、审核状态、运算状态），以便快速定位单据。
2. 作为跟单员，我想按 PI 号、系统单号、客户、销售日期区间筛选订单，以便对账与跟进。
3. 作为跟单员，我想区分「在册」与「回收站」列表，以便处理误删单据。
4. 作为跟单员，我想在列表上看到 **运算状态**（已运算/未运算），以便知道是否可供仓库/采购订料。

### 新建与编辑主表

5. 作为跟单员，我想新建未审核销售订单并手填 **PI 号**，以便与外贸业务习惯一致。
6. 作为跟单员，我想在首次保存时获得 **系统单号**（`PI-YYYYMMDD-XXX`），以便内部流水追溯。
7. 作为跟单员，我想选择 **销售日期**（必填）和可选 **交货日期**（不得早于销售日期），以便排产与交期管理。
8. 作为跟单员，我想从已审核的销售客户主档选择 **客户代码**，并自动带出 **客户名称快照**，以便单据自洽且不受后续客户改名影响。
9. 作为跟单员，我想从 `UB_ERP_System_currency` 下拉选择币别（人民币/美元/欧元/港元），并保存 **币别代码与名称快照**，以便历史订单不受币别表变更影响。
10. 作为跟单员，我想填写 **备注** 和 **小数点位数**（默认 2，一期无行金额），以便后续金额功能沿用。
11. 作为跟单员，我想在保存后 **不可再改 PI 号**，以便 PI 作为全库关联键稳定。
12. 作为跟单员，我想在编辑未审单时修改销售日期但 **系统单号不变**，以便单号与首次保存日一致。
13. 作为系统，我需要在保存时校验 PI 号 **全表唯一**（含已软删），以便避免关联混乱。
14. 作为系统，我需要在 PI 号仅在被 **彻底删除** 该 PI 全部关联数据后才允许复用，以便软删订单仍占用 PI 号。

### 销售订单明细

15. 作为跟单员，我想在明细 Tab 为每行选择 **货品编码**（`bom_000.kcaa01`）并录入 **订货数量**（>0），以便表达订货需求。
16. 作为跟单员，我想看到货品名称、规格、颜色、单位等 **只读展示字段**（来自 `bom_000` 快照），以便录单时不手填冗余信息。
17. 作为系统，我需要在保存时将 **同一 PI 下同一货品编码** 的多行 **合并为一行、数量相加**，以便明细与 PI BOM 一款一行一致。
18. 作为跟单员，我想在明细 Tab 对每行使用 **操作**（如同步 BOM），以便按款维护 BOM。
19. 作为跟单员，我想在删除明细行前收到二次确认（说明保存后才落库），以便避免误删。
20. 作为跟单员，我想最多录入 **999 行** 明细（合并前），以便符合业务上限。

### 保存与 PI BOM 对齐

21. 作为跟单员，我想点击 **保存（提交）** 写入主从表并在同一事务内 **对齐 PI BOM**，以便订单与 BOM 结构一致。
22. 作为系统，我需要在保存时 **物理删除** 已不在明细中的款对应的 `UB_ERP_Bom_Sales` 头及其全部 `UB_ERP_Bom_Sales_list`，以便删 TEST1 后 TEST1 的子件全部消失。
23. 作为系统，我需要在保存时 **不** 用主 BOM 覆盖仍在单上且已有 PI BOM 的款，以便用户在 PI 内改的用量（如 0.1→0.2）不被偷偷改回。
24. 作为系统，我需要在明细 **首次出现** 某 `kcaa01` 时从主 BOM **建立** 该款 PI BOM（头+子件，展开≤4层），以便新款自动有结构。
25. 作为系统，我需要在某款 **删掉后又加回同一货品编码** 时从主 BOM **重建** 该款 PI BOM，以便视为新款。
26. 作为系统，我需要在保存变更明细货品集合或 **仅改订货数量** 后将订单标为 **未运算**，以便禁止下游使用过期物料单。
27. 作为系统，我需要在保存时 **不写入** 物料单表，以便物料单仅来自一键运算。

### 同步 BOM

28. 作为跟单员，我想对指定明细行点击 **同步 BOM**，以便用当前主 BOM 覆盖该款 PI BOM。
29. 作为系统，我需要在同步 BOM 后将订单标为 **未运算**，以便用户知悉须重新运算。
30. 作为系统，我需要在未点同步 BOM 且无例外时 **禁止** 任何操作从主 BOM 覆盖已有 PI BOM 内容，以便 PI 内手工改动能保留。

### 一键运算与物料单

31. 作为跟单员，我想执行 **一键运算**，以便生成/更新全系统订料口径的物料单。
32. 作为系统，我需要在一键运算时 **只读 PI BOM**，**禁止** 从主 BOM 偷拉，以便与「同步 BOM」职责分离。
33. 作为系统，我需要将运算结果写入 `UB_ERP_Bom_pi_cost`（同 BOM 成本运算用量表语义）和 `UB_ERP_Bom_pi_consumption`（按子件编码+备注合并），以便下游统一口径。
34. 作为系统，我需要在运算写入时 **不乘订货数量**，以便结构用量与订单数量解耦。
35. 作为跟单员，我想在备料展示中看到 **用量 × 该款订货数量**，以便按款算总需求。
36. 作为跟单员，我想在订单 **未运算** 时一键运算 **重写整单** 物料单（当前明细全部在单款），以便改数量或删款保存后一次算清。
37. 作为跟单员，我想在订单 **已运算** 且仅对部分行 **同步 BOM** 后，一键运算 **只重算本次同步过的款** 的物料单，以便其它款物料单不被改动。
38. 作为系统，我需要在运算成功后把订单标为 **已运算**，以便仓库/采购/出入库可引用。
39. 作为系统，我需要在 BOM **循环引用** 或展开超 **4 层** 时使同步/运算失败并提示 **货品编码**，以便用户修正主数据。

### 审核与生命周期

40. 作为主管，我想 **审核** 未审订单，以便锁定单据。
41. 作为主管，我想 **反审** 已审订单，以便退回修改。
42. 作为跟单员，我想在 **已审核** 状态下无法编辑主从、无法软删，以便已审数据稳定。
43. 作为系统，我需要在审核/反审时 **不** 自动同步主 BOM、**不** 自动运算物料单，以便状态变更与 BOM/运算解耦。
44. 作为跟单员，我想 **软删** 仅未审订单并在回收站 **恢复**，以便误操作可逆。
45. 作为跟单员，我想在回收站内对未审订单 **彻底删除**，以便释放 PI 号并清除全部关联数据。
46. 作为系统，我需要在彻底删除时按 PI 号 DELETE 订单主从、PI BOM、物料单全部表，以便无孤儿数据。
47. 作为系统，我需要在已审订单上禁止软删与彻底删（须先反审），以便符合主从单据规范。

### 权限、审计与操作人

48. 作为管理员，我想通过菜单权限控制销售订单的增删改查、审核、运算等操作，以便职责分离。
49. 作为审计员，我想在全局操作审计中看到保存、审核、运算等 **中文可读** 日志，以便追溯。
50. 作为系统，我需要在新增/保存时写入 `uid`、`uname`（登录 usercode）、`utruename`、`addtime`/`edittime` 及客户端 IP，且 **禁止** 把 `Sys_Users.uname` 当登录账号，以便与采购报价一致。
51. 作为系统，我需要在子表 INSERT 时由服务端写入操作人字段（若列存在），**禁止** 前端 body 覆盖，以便防篡改。

### 下游与展示（一期边界内）

52. 作为仓库员，我想在订单 **已运算** 后依据物料单订料，以便与全系统口径一致。
53. 作为系统，我需要在订单 **未运算** 时让下游 **禁止** 引用物料单，以便避免按过期结构订料。
54. 作为跟单员，我想在 PI BOM 界面查看/编辑某款的子件结构（在权限与未审前提下），以便按 PI 调用量。
55. 作为跟单员，我想查看物料单明细与汇总（运算后），以便核对备料。

### 校验与错误提示

56. 作为跟单员，我想在 PI 号为空、重复、销售日期为空、明细缺编码或数量非法时保存失败并看到明确提示，以便一次改对。
57. 作为跟单员，我想在当日系统单号序号已达 999 时保存失败并提示改日或协调，以便不破坏单号格式。
58. 作为跟单员，我想在选择未审核或已删除客户时保存失败，以便客户主数据质量可控。

### 系统单号

59. 作为系统，我需要在首次保存按 **销售日期** 生成 `PI-YYYYMMDD-001`～`999` 流水，以便按自然日重置序号。
60. 作为系统，我需要在同日取已有系统单号最大序号 +1，以便不重复。

## Implementation Decisions

### 模块划分（建议深模块）

| 模块 | 职责 | 可测接口 |
|------|------|----------|
| **销售订单持久化** | 主从 CRUD、合并明细、PI 号/系统单号/校验、pass/del、运算状态列 | 纯函数：合并键、校验器、单号生成 |
| **PI BOM 对齐器** | 保存时按款 diff 明细与 `UB_ERP_Bom_Sales`；删款物理删；新款/再加码从主 BOM 建款（≤4层） | 输入：明细 kcaa01 集合 + 现有 PI BOM → 删/建计划 |
| **主 BOM 门禁** | 判断是否允许主 BOM 写入；同步 BOM 执行覆盖 | 三种例外 + 同步标志 |
| **BOM 展开服务** | 复用库存 BOM 展开（`bom_000`/`Bom_parts`），深度限制与循环检测 | 与现有 `bomUsageTreeBuild` 同类契约 |
| **同步 BOM 执行器** | 按行 kcaa01 从主 BOM 覆盖 `UB_ERP_Bom_Sales*`；标未运算 | 单款事务片段 |
| **一键运算引擎** | 读 PI BOM，写 `pi_cost`/`pi_consumption`；整单/部分款两种范围；标已运算 | 与 BOM 成本运算用量表对齐的独立服务 |
| **物料单读写** | 按 PI 查询 cost/consumption；供下游 API（本期可实现查询，订料集成可后续） | 按 PI 聚合 |
| **销售订单 API 层** | REST：list/get/post/put、审核/反审/软删/恢复/彻底删、sync-bom、calculate | 与采购报价路由风格一致 |
| **销售订单 UI** | 列表 + 大弹窗（主表 Tab / 明细 Tab）；明细操作列；运算/同步按钮；已审锁 | 对齐采购报价交互 |

### 数据模型（已定）

- **关联键**：全模块以 **PI 号** 串联订单、PI BOM、物料单；**系统单号** 仅展示/检索。
- **主表**：客户/币别存 **代码 + 名称快照**；运算状态（如 `isok`）实现时列名探测。
- **明细**：`kcaa01` + 订货数量 + `bom_000` 展示快照；合并规则：同 PI + 同 `kcaa01` 数量相加。
- **PI BOM**：一头一行成品（`kcaa01`）；`UB_ERP_Bom_Sales_list` 子件结构同 `Bom_parts`；**禁止** 整 PI DELETE ALL + INSERT。
- **物料单**：仅一键运算写入；`pi_consumption` = `pi_cost` 按子件编码+备注合并；运算不含订货数量。

### 保存流水线（事务内顺序）

1. 提交校验（PI、日期、明细、客户、币别等）
2. 主表 INSERT/UPDATE（新增写 PI + 系统单号；编辑不改 PI/系统单号）
3. 订单明细整批替换（允许 DELETE 旧 list + INSERT 新 list，与金标准一致）
4. PI BOM 对齐（按款删/建，非整 PI 重插）
5. 若明细 kcaa01 集合或订货数量变化 → 主表 **未运算**

### 主 BOM 门禁（三种允许写入）

| 场景 | 行为 |
|------|------|
| 用户 **同步 BOM**（按行） | 主 BOM → 覆盖该款 PI BOM；未运算 |
| 例外 1：删款后同码再加 | 视为新款，主 BOM 重建该款 |
| 例外 2：首次入单该 kcaa01 | 主 BOM 建立该款 |
| 其它 | 保存/运算均不得用主 BOM 改已有款内容 |

### 一键运算范围

- **未运算**：对当前明细 **全部在单款** 按 PI BOM **重写整单** `pi_*`。
- **已运算 + 部分同步**：仅重算 **本次同步过的款** 的 `pi_*`；其它款禁止改动。

### API 契约（建议，与金标准对齐）

- `GET /api/sales-order/list` — 分页列表（含 pass/del/运算状态筛选）
- `GET /api/sales-order/:id` — 主表 + 明细
- `POST /api/sales-order` — `{ header, lines[] }` 新增保存
- `PUT /api/sales-order/:id` — 保存（含 PI BOM 对齐）
- `POST /api/sales-order/:id/approve` / `unapprove`
- `POST /api/sales-order/:id/soft-delete` / `restore` / `hard-delete`（回收站且未审）
- `POST /api/sales-order/:id/sync-bom` — body `{ lineIds[] }` 或 `{ kcaa01 }` 按行同步
- `POST /api/sales-order/:id/calculate` — 一键运算；body 可带 `syncedKcaa01[]` 区分部分重算
- `GET /api/sales-order/:id/pi-bom` — 按款查 PI BOM（可选）
- `GET /api/sales-order/:id/material-bill` — 查物料单（已运算时有效）

权限：`apiPermissionGate` 登记；`action_map` 审计文案含 PI 号、明细款数、运算范围等。

### 前端交互（已定）

- 明细 Tab：**货品编码** + **订货数量** 可编辑；其余只读来自 `bom_000`。
- 每行 **操作**：同步 BOM。
- 工具栏/页脚：**一键运算**；展示运算状态。
- 已审：主从锁定，提示先反审（对齐采购报价文案模式）。
- 币别下拉：**读 `UB_ERP_System_currency` 全表**，禁止写死 001～004。

### 复用与依赖

- 主从单据模式、审计、pass/del：**采购报价** 金标准。
- BOM 展开、循环检测、用量语义：**库存 BOM 资料** 现有服务（`bomUsageTreeBuild`、`bomUsageYl` 等）。
- 客户主档：`UB_ERP_System_sales_customer`（须已审、未删）。
- 货品主档：`bom_000`。
- SQL Server 2008 R2：`ROW_NUMBER()` 分页，禁止 OFFSET-FETCH。

### 状态机（主表，简化）

```
[新建 pass=0, del=0, 未运算]
  → 保存 → 可能对齐 PI BOM，可能→未运算
  → 一键运算 → 已运算
  → 改明细/数量/同步BOM → 未运算
  → 审核 → pass=1（锁定编辑）
  → 反审 → pass=0
  → 软删 → del=1（回收站）
  → 彻底删除 → 物理清除 + PI 可复用
```

## Testing Decisions

### 原则

- 测 **对外行为**（API 状态码、库表行数、运算状态、门禁是否生效），不测内部函数名或私有步骤顺序。
- 集成测试优先：**保存 + PI BOM 对齐 + 运算** 端到端；单元测试覆盖 **合并、校验、单号生成、门禁判断、运算范围选择**。

### 建议覆盖模块

| 模块 | 测试类型 | 先验样例 |
|------|----------|----------|
| 提交校验 | 单元 | 采购报价校验测试（若有） |
| 明细合并 | 单元 | — |
| 系统单号流水 | 单元 | 同日 999 上限、跨日重置 |
| PI BOM 对齐 | 集成 | 删 TEST1 后仅余 TEST2 头+list；新款建头 |
| 主 BOM 门禁 | 集成 | 改用量不同步→保存后仍为 0.2；同步后变主 BOM |
| 未运算触发 | 集成 | 仅改数量→未运算；下游读物料单应拒绝或空 |
| 一键运算 | 集成 | 未运算整单重写；已运算部分同步只改同步款 |
| 审核锁 | 集成 | pass=1 时 PUT 400 |
| 软删/彻底删 | 集成 | 软删 PI 不可复用；彻底删后可复用 PI |
| 循环/超深 BOM | 集成 | 与库存 BOM 运算失败提示一致 |

### 测试数据

## Out of Scope

- **一期不做**：明细行单价、行备注、行交期、行金额计算；小数位仅主表配置存库。
- **不在保存时** 自动生成或刷新物料单。
- **不在审核/反审/保存时** 自动同步主 BOM 或自动一键运算。
- **不** 对 PI BOM 做「整 PI 先 DELETE 全部再 INSERT」。
- **不** 修改纸格导入解析冻结文件。
- **下游模块改造**（仓库/采购/出入库实际读物料单接口）— 本期可交付物料单查询与运算状态，订料集成单列需求。
- **PI 号保存后修改**、软删后 PI 号复用（须彻底删）。
- **系统单号** 随销售日期变更而改前缀（编辑不改号）。
