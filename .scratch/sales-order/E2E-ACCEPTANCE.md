# 销售订单 — 端到端验收清单

> 与 `CONTEXT.md` 第七节、`PRD.md` 业务示例 **PI-002** 对齐。执行前请确认：API `3001`、前端 `5173`、测试账号具备菜单 `supply-chain/daily/sales-order` 的 `view`/`add`/`edit`/`audit`/`delete` 权限。

**自动化**：`npm run test:sales-order`（需 `.env` 中 `DB_*`、`E2E_USERCODE`、`E2E_PASSWORD`）；UI 冒烟 `npm run e2e:sales-order`（issue 01）。

---

## A. 环境与权限

- [√] 登录后进入 **供应链 → 日常工作 → 销售订单**
- [√] 列表可加载；无 403
- [ ] 全局操作审计中可见销售订单相关中文动作（保存、同步 BOM、一键运算、PI BOM 等）

## B. 录单与 PI BOM 对齐（issue 02）

- [ ] **新增**：填写唯一 PI 号、销售日期、在册已审客户、币别；明细选 **≥2 款**（验收用 `BAG-TEST1`、`BAG-TEST2`，或库内等效成品编码）
- [ ] 保存成功；生成系统单号 `PI-YYYYMMDD-XXX`；列表显示 **未运算**
- [ ] 再次打开：明细款与 `UB_ERP_Bom_Sales` 头数一致（每款一头）
- [ ] **删一款**（如 TEST1）后保存：该款 PI BOM 头及 `UB_ERP_Bom_Sales_list` 全删；订单 **未运算**
- [ ] **加回同码** 保存：从主 BOM 重建该款 PI BOM（例外 1）

## C. PI 内改用量，保存不覆盖（issue 06 + CONTEXT 主 BOM 门禁）

- [ ] 打开 **PI BOM** Tab，选 **TEST2**，将某子件用量 **0.1 → 0.2**，**保存 PI BOM**
- [ ] 订单变为 **未运算**；刷新树仍为 **0.2**
- [ ] **保存订单**（不改货品行/数量）：PI BOM 仍为 **0.2**（非同步、非删款再加则主 BOM 不得覆盖）

## D. 按行同步 BOM（issue 04）

- [ ] 明细 Tab 对 **TEST2** 点 **同步 BOM**；确认后 PI BOM 恢复为主 BOM 用量（如 **0.1**）
- [ ] 订单 **未运算**；**TEST1** 未同步则其 PI BOM 与改前一致

## E. 一键运算与物料单（issue 05）

- [ ] 订单 **未运算** 时 **一键运算**：成功；状态 **已运算**
- [ ] **物料单** Tab 可查看明细与汇总；备料量 ≈ **用量 × 该款订货数量**
- [ ] **未运算** 时打开物料单：提示无效或 409
- [ ] **已运算** 且仅同步过 TEST2 后再次运算（body 带 `syncedKcaa01: ['BAG-TEST2']` 或 UI 等价流程）：**仅 TEST2** 物料单变化；**TEST1** 物料单行不变

## F. 改订货数量（issue 02 运算状态）

- [ ] 仅将某款 **订货数量** 调大/调小后保存：订单 **未运算**；须再运算后物料单备料量随数量变

## G. 审核 / 回收站（issue 03）

- [ ] **审核** 后：不可保存订单、不可改 PI BOM、不可同步 BOM、不可软删
- [ ] **反审** 后可再编辑 PI BOM
- [ ] **软删** 进回收站；**同一 PI 号不可新建**（唯一约束含软删）
- [ ] **恢复** 后仍在册
- [ ] 回收站 **彻底删除**（未审）：PI 下 `Sales_order*`、`Bom_Sales*`、`Bom_pi_*` 清空；**PI 号可复用**

## H. PI-002 联调摘要（与 PRD 一致）

| 步骤 | 预期 |
|------|------|
| PI 内 TEST2 子件 0.1→0.2，未同步 | 保存订单后仍为 0.2 |
| 同步 TEST2 | 主 BOM 盖回；**未运算** |
| 一键运算 | 只读 **PI BOM**，不偷拉主 BOM |
| 已运算后仅同步 TEST2 再运算 | 只重算 TEST2 物料单 |
| 删 TEST1 保存 | Sales 仅余 TEST2；整单 **未运算** |
