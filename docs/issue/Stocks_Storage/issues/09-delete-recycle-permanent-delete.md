# 09 - 软删除、回收站、恢复与彻底删除

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage/PRD.md`

## What to build

交付入库单删除和恢复闭环：待审核入库单可以软删除进入回收站，回收站中可以查看和恢复；超级管理员可以彻底删除符合条件的记录。已审核入库单不能直接删除，必须先反审核。

本工单要确保删除动作不会让已审核库存记录绕过反审核直接消失。

## Acceptance criteria

- [ ] 只有 `pass=0`、`del=0`、`sp_flag=0`、`closed=0` 且非类型 `8` 的入库单可软删除。
- [ ] 已审核入库单删除按钮不可用或后端拒绝，并提示必须先反审核。
- [ ] 软删除设置 `del=1` 并写入删除人和删除时间。
- [ ] 软删除写入操作日志。
- [ ] 回收站在当前模块内切换或筛选，不做旧系统独立恢复页。
- [ ] 回收站只处理已软删除的待审核单据。
- [ ] 回收站可打开只读详情。
- [ ] 恢复设置 `del=0` 并写入操作日志。
- [ ] 彻底删除仅超级管理员可操作。
- [ ] 彻底删除仅允许待审核、已删除、未复核、未结案、非类型 `8` 的入库单。
- [ ] 普通 `delete` 权限用户不能彻底删除。
- [ ] 若异常旧数据中已审核单据进入回收站，可查看或恢复，但不能彻底删除。
