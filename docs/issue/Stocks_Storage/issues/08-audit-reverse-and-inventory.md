# 08 - 审核、反审核与库存统计口径

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage/PRD.md`

## What to build

交付入库单正式影响库存的生命周期闭环：待审核单据可以审核，审核后主表和明细同步变为已审核，并进入库存实时汇总；已审核单据可以反审核，反审核后回到待审核并退出库存统计。

本工单不做审核不通过，也不做财务复核流程。

## Acceptance criteria

- [ ] 只有 `pass=0`、`del=0`、`sp_flag=0`、`closed=0` 且非类型 `8` 的入库单可审核。
- [ ] 审核后主表和明细 `pass` 同步改为 `1`。
- [ ] 审核写入审核人和审核时间。
- [ ] 审核写入操作日志。
- [ ] 只有 `pass=1`、`del=0`、`sp_flag=0`、`closed=0` 且非类型 `8` 的入库单可反审核。
- [ ] 反审核后主表和明细 `pass` 同步改回 `0`。
- [ ] 反审核写入操作日志。
- [ ] 第一版页面不出现“审核不通过”入口。
- [ ] 已审核且未删除的明细 `kcao03` 进入库存增加统计。
- [ ] 待审核、已删除、已反审核记录不进入库存增加统计。
- [ ] 库存统计维度为物料编码、仓库编码、颜色、版本、库位；空库位按空值汇总。
- [ ] 第一版不新增库存余额表。
