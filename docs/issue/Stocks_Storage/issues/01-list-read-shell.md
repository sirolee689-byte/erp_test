# 01 - 入库单列表、搜索与只读详情

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage/PRD.md`

## What to build

交付入库单模块的第一条可演示路径：用户能进入入库单页面，看到真实入库单列表，按条件搜索和分页，并能打开一张入库单查看主表和明细的只读详情。

本工单要打通前后端最小闭环：模块入口、权限动作、列表接口、搜索接口、分页、单笔详情接口、前端列表页、只读详情弹窗。详情只展示，不提供新增、编辑、审核、删除、打印。

## Acceptance criteria

- [ ] 入库单页面不再是占位页，页面形态与外协订单模块一致：列表为主，顶部有搜索和操作区。
- [ ] 默认列表显示 `del=0` 的入库单，支持分页。
- [ ] 搜索条件至少支持入库单号 `kcan01`、入库类型 `kcan03`、仓库 `kcan06`、关联方、关联单号 `kcan04`、审核状态 `pass`、入库日期 `kcan02` 范围、纸质单号 `kcan08`。
- [ ] 列表列至少显示入库单号、入库日期、入库类型、仓库、关联方、关联单号、审核状态、经手人、创建人、备注。
- [ ] 单笔详情接口返回主表一行和明细数组。
- [ ] 详情弹窗只读展示主表信息、明细行、审核/删除/复核/结案状态。
- [ ] `sp_flag=1`、`closed=1`、类型 `8` 的旧数据能正常查看，并显示不可操作含义。
- [ ] 类型 `8` 在详情或列表操作处显示不可操作标志。
- [ ] 无数据、接口失败、无权限时有清晰提示，不出现空白页。
