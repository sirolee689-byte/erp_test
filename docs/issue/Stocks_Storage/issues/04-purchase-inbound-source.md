# 04 - 采购入库关联采购单批量添加

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage/PRD.md`

## What to build

交付采购入库的端到端路径：用户选择供应商和采购单，从采购单批量带入可入库明细，系统按采购单剩余可入库数量严控超量，保存成待审核入库单。

本工单只做采购入库一种关联型入库，作为后续外协、生产、销售退货来源接入的样板。第一版只读取和校验采购单，不反写采购单已入库数量。

## Acceptance criteria

- [ ] 新增页支持入库类型 `1` 采购入库。
- [ ] 采购入库必须选择供应商和采购单。
- [ ] 服务端校验采购单已审核、未结案、未删除，并且供应商与当前入库供应商一致。
- [ ] 供应商名称保存时由服务端按供应商编码重新查询并写入快照 `kehu`。
- [ ] 批量添加窗口只展示当前采购单可入库的采购明细。
- [ ] 批量添加带回材料快照、采购单价、关联订单明细键 `kcao02`、关联单号 `kcan04` 和可入库数量上限。
- [ ] 采购入库不允许手工新增无采购来源的物料行。
- [ ] 保存时后端再次校验采购单与供应商关系。
- [ ] 保存时后端按实时可入库数量阻止超量入库。
- [ ] 超量时给出明确提示，第一版不做特殊供应商豁免。
- [ ] 保存成功后采购入库为待审核，库存不变化。
- [ ] 保存成功写入操作日志。
