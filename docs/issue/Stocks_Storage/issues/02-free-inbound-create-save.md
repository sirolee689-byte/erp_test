# 02 - 其他入库与盘盈入库新增保存

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage/PRD.md`

## What to build

交付无上游来源的入库保存闭环：用户能新增“其他入库”和“盘盈入库”，选择仓库，填写自由文本关联单位、纸质单号、备注，手工选择物料明细，保存成待审核入库单。

本工单要完成后端最终生成入库单号、写入主表和明细、写物料快照、写操作日志，并在前端保存后回到待审核可查状态。库存暂不因待审核保存而变化。

## Acceptance criteria

- [ ] 新增页支持入库类型 `0` 其他入库和 `7` 盘盈入库。
- [ ] `kcan01` 由后端在保存时最终生成，前端只展示建议号且不可编辑。
- [ ] 入库单号日期部分按保存当天生成，不跟随用户填写的入库日期。
- [ ] 入库日期 `kcan02` 默认当前时间，保存前可修改。
- [ ] 仓库必填，保存时服务端按仓库编码重新查询并写入仓库名称快照 `ck`。
- [ ] 其他入库和盘盈入库允许自由文本关联单位，并与编码型关联方字段区分。
- [ ] 明细通过手工物料选择录入，至少需要一条有效明细。
- [ ] 明细数量 `kcao03` 必须大于 0，保存时 `kcao031` 与 `kcao03` 相同。
- [ ] 服务端按物料编码重新查询物料主档，写入已有 `kcaa01` 到 `kcaa35` 物料快照字段。
- [ ] 前端备注或 info 正确写入明细数据库字段 `Describe`。
- [ ] `location` 保存真实库位，不写入版本号。
- [ ] 保存成功后单据为 `pass=0`、`del=0`、`closed=0`、`sp_flag=0`，并可在列表中查到。
- [ ] 保存成功写入操作日志。
