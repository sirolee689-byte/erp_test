# 03 - 待审核入库单编辑与金额计算

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage/PRD.md`

## What to build

交付待审核入库单的编辑闭环，并补齐第一版统一金额计算规则。用户能编辑未审核、未复核、未结案的无来源入库单，修改入库日期、仓库、备注、明细数量、单价、税点和备注，保存后明细采用整批重写。

本工单重点把“可编辑”和“不可编辑”的边界落稳：已审核、已复核、已结案、类型 `8` 均不能编辑。

## Acceptance criteria

- [ ] 只有 `pass=0`、`del=0`、`sp_flag=0`、`closed=0` 且非类型 `8` 的入库单能进入编辑。
- [ ] 已审核、已复核、已结案、类型 `8` 的入库单前后端都禁止保存修改。
- [ ] 待审核入库日期 `kcan02` 可修改，已审核或已复核时只读。
- [ ] 编辑保存明细采用整批重写，不做行级差异更新。
- [ ] 编辑保存后仍为 `pass=0`，可在待审核状态中查看。
- [ ] 不含税模式 `in_tax=2` 时税点 `tax` 只能为 `0`。
- [ ] 以不含税单价 `kcao04` 和税点 `tax` 为主要输入，自动计算含税单价 `kcao041`、不含税金额 `kcao05`、含税金额 `kcao051`。
- [ ] 用户直接修改含税单价 `kcao041` 时，页面反算不含税单价并刷新两套金额。
- [ ] 数量变化后金额自动重新计算。
- [ ] 保存成功写入编辑操作日志。
