# 02 - 其他出库与盘亏出库保存

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage_out/PRD.md`

## What to build

交付无来源出库的新增保存闭环：用户可以新增 `0` 其他出库和 `9` 盘亏出库，从当前仓库可用库存中选择明细，保存为待审核出库单，并立即占用可用库存。

本工单同时建立出库单库存可用数量计算的基础能力：可用库存 = 已审核入库 - 已审核出库 - 未审核出库占用，并按物料、仓库、颜色、版本、库位维度计算。

## Acceptance criteria

- [ ] 新增出库单保存时由服务端生成最终 `kcap01`，用户不能手工修改。
- [ ] 新增保存后一律为 `pass=0`，不支持自动审核。
- [ ] 新增默认 `del=0`、`closed=0`、未复核。
- [ ] 出库日期 `kcap02` 默认当前时间，待审核可编辑。
- [ ] 批量添加前必须选择出库类型、是否含税、仓库，缺失时有明确提示。
- [ ] `0` 其他出库和 `9` 盘亏出库只能从当前仓库有可用库存的物料中选择。
- [ ] 同一物料在不同颜色、版本、库位上的库存行分开选择和扣减。
- [ ] 保存时服务端重新校验库存可用数量，禁止超过可用库存。
- [ ] 保存待审核后，该单明细计入未审核出库占用。
- [ ] 盘亏出库不强制关联单位；其他出库允许自由文本关联单位但要和编码型关联方区分。
- [ ] 服务端按物料编码重查物料主档并写入存在的 `kcaa01` 到 `kcaa35` 快照字段。
- [ ] 物料不存在或已删除时禁止保存新单。
