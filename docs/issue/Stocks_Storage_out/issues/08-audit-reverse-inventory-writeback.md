# 08 - 审核、反审核、库存与来源反写

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/Stocks_Storage_out/PRD.md`

## What to build

交付出库单正式影响库存和来源单据的生命周期闭环：待审核出库单可以审核，审核后主表和明细同步为已审核，未审核占用转为正式出库，并按出库类型反写上游来源单据；已审核单据可以反审核，反审核后回到待审核占用，并反向扣减来源累计数量。

本工单必须保证库存和来源反写在同一业务事务内完成，不能出现库存已扣但来源未更新的半成品状态。

## Acceptance criteria

- [ ] 只有 `pass=0`、`del=0`、未复核、`closed=0` 且类型不是 `7`、`8`、`10` 的出库单可审核。
- [ ] 审核后主表和明细 `pass` 同步改为 `1`。
- [ ] 审核写入审核人和审核时间。
- [ ] 审核写入操作日志。
- [ ] 审核前重新校验库存可用数量和来源剩余数量。
- [ ] 审核后未审核占用转为正式出库。
- [ ] `0` 其他出库和 `9` 盘亏出库不做来源反写。
- [ ] `1` 采购退货审核反写采购单明细 `kcak07`，并按单位换算规则写入。
- [ ] `2` 外协领料和 `3` 外协退货审核反写外协明细 `wxak08`，并按单位换算规则写入。
- [ ] `4` 生产领料审核反写派工明细 `scak04`，直接使用出库数量。
- [ ] `5` 生产返修审核反写派工明细 `scak05`，直接使用出库数量。
- [ ] `6` 成品出库审核反写销售订单明细 `xsak06`，直接使用出库数量。
- [ ] 只有 `pass=1`、`del=0`、未复核、`closed=0` 且类型不是 `7`、`8`、`10` 的出库单可反审核。
- [ ] 反审核后主表和明细 `pass` 同步改回 `0`。
- [ ] 反审核后正式出库退回未审核占用，不释放库存占用。
- [ ] 反审核反向扣减来源累计数量，且不能把来源累计数量扣成负数。
- [ ] 反审核必须重新校验库存和来源累计数量，不能导致库存可用数量或来源累计数量小于 0。
- [ ] 第一版页面不出现“审核不通过”入口。
