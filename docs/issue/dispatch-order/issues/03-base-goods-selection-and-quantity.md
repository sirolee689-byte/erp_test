# 03 - 本厂和大板派工选货与可派工数量

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/dispatch-order/PRD.md`

## What to build

交付本厂派工和大板派工的批量选货路径：用户在一张已选定 PI 的派工单里打开选货弹窗，从已审核、未删除、未关闭的销售订单明细中选择货品，系统展示销售数量、已派工数量、可派工数量、已入库数量、返修数量，并把选中的货品快照回填到派工单明细。

本工单要实现可派工数量的基础规则：`销售数量 xsak03 - 非删除派工明细 scak03 汇总`。未审核派工单也计入，已删除派工单不计入。编辑场景排除当前派工单可以先提供服务参数，正式保存拦截由后续工单完成。

本工单不做委外派工特殊统计规则、不做整单保存校验、不做审核。

## Acceptance criteria

- [ ] 本厂派工和大板派工选货弹窗从 `UB_ERP_Sales_order_list` 读取关联 PI 下的货品。
- [ ] 只展示主单满足 `del=0`、`closed=0`、`pass=1` 的销售订单货品。
- [ ] 弹窗支持搜索和分页，默认每页 10 条。
- [ ] 弹窗展示销售数量、已派工数量、可派工数量、已入库数量、返修数量。
- [ ] 可派工数量小于等于 0 的货品显示为不可选，按钮置灰，并有明确提示。
- [ ] 已派工数量统计不区分审核状态，只排除 `del=1` 的派工单。
- [ ] 选中货品后，明细 `scak03` 默认等于可派工数量，`scak04` 写入已派工数量快照，`scak05` 写入返修数量快照。
- [ ] 明细快照带入 `pi`、`version`、`kcaa01`、`kcaa02`、`kcaa03`、`kcaa04`、`kcaa05`、`kcaa06`、`kcaa09`、`kcaa10`、`kcaa11`、`kcaa12`、`kcaa13`、`kcaa14`、`kcaa15`、`systemcode`、`GUID` 等 PRD 第一版字段。
- [ ] 同一派工单里已经选择过的 `kcaa01` 不能再次选择。
