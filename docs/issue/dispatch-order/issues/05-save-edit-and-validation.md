# 05 - 派工单明细保存、编辑与后端校验

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/dispatch-order/PRD.md`

## What to build

交付派工单新增和编辑的正式保存闭环：用户从选货弹窗带回明细后，可以填写或调整本次派工数量 `scak03`，保存时后端写入主表和明细，并统一做必填、明细数量、重复货品、单 PI、超额派工校验。

编辑只允许未审核、未删除单据。保存后派工类型、关联 PI、供应商、生产车间变成只读；可改派工日期、交货日期、备注；明细可新增、删除，已有明细只能改 `scak03`。编辑保存采用旧系统方式：删除旧明细，再插入当前页面明细。

本工单不做审核、反审核、回收站。

## Acceptance criteria

- [ ] 正式新增保存必须同时写入主表、N 条明细和一条操作日志。
- [ ] 保存时要求派工日期、生产车间、派工类型等必填字段有效。
- [ ] 保存时至少需要一条明细；编辑时页面可临时删空，但最终保存必须拦截空明细。
- [ ] 明细 `scak03` 必须大于 0。
- [ ] 同一派工单内不能出现重复 `kcaa01`。
- [ ] 同一派工单内所有明细必须属于同一个 PI。
- [ ] 后端按实时可派工数量拒绝超额派工，不依赖 `scak04` 快照。
- [ ] 新增计算可派工数量时统计所有匹配的非删除派工单；编辑计算时排除当前派工单。
- [ ] 编辑只允许 `pass=0` 且 `del=0` 的派工单，已审核单据前后端都不能保存修改。
- [ ] 保存后的派工单仍为 `pass=0`，保存成功回到未审核列表。
- [ ] 已有明细的货品快照字段只读，只有 `scak03` 可改。
