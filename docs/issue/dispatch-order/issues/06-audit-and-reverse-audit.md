# 06 - 派工单审核与反审核

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/dispatch-order/PRD.md`

## What to build

交付派工单的审核和反审核生命周期：未审核派工单审核后成为正式生产指令，主表和明细状态一起变为已审核；已审核派工单第一版允许直接反审核，主表和明细一起回到未审核，方便后续修改。

第一版不做基于成品入库或生产进度的反审核限制，因为派工单号是否能直接关联成品入库仍待确认。

本工单不做软删除、回收站、打印、作废。

## Acceptance criteria

- [ ] 只有 `pass=0` 且 `del=0` 的派工单显示并允许审核。
- [ ] 审核时主表 `pass` 改为 `1`，写入当前用户 `passuid` 和 `passuname`。
- [ ] 审核时所有明细同步改为 `pass=1`，写入同一审核人信息。
- [ ] 审核成功后页面回到或刷新到已审核列表。
- [ ] 已审核派工单详情只读，不能编辑保存。
- [ ] 只有 `pass=1` 且 `del=0` 的派工单显示并允许反审核。
- [ ] 反审核时主表和明细 `pass` 改回 `0`，并清空或按项目规则处理 `passuid`、`passuname`。
- [ ] 反审核成功后页面回到或刷新到未审核列表。
- [ ] 审核和反审核都写入项目通用操作日志。
