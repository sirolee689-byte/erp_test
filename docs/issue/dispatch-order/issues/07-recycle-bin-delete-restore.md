# 07 - 派工单删除、回收站、恢复与彻底删除

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/dispatch-order/PRD.md`

## What to build

交付派工单删除和回收站闭环：未审核派工单可软删除进入回收站，回收站中可查看、恢复，也可对未审核记录执行彻底删除。已审核单据受到保护，必须先反审核才能删除或彻底删除。

本工单按项目通用回收站规则实现。普通删除只改主表删除标记；彻底删除在回收站内物理删除主表和关联明细。

本工单不做作废、不做关闭、不做打印。

## Acceptance criteria

- [ ] 普通列表中只有 `pass=0` 且 `del=0` 的派工单允许软删除。
- [ ] 软删除时主表设置 `del=1`，并写入 `delid`、`delname`、`deltruename`、`deltime`。
- [ ] 已审核派工单不能软删除，前端按钮隐藏或置灰，后端也拒绝。
- [ ] 回收站只显示 `del=1` 的派工单。
- [ ] 回收站支持查看详情，详情只读。
- [ ] 回收站支持恢复，恢复时主表 `del` 改回 `0`。
- [ ] 回收站支持彻底删除，但只允许 `pass=0` 的记录。
- [ ] 彻底删除时物理删除主表和 `scak01` 对应的明细。
- [ ] 若历史异常数据在回收站中仍为 `pass=1`，可查看或恢复，但不能彻底删除。
- [ ] 删除、恢复、彻底删除均写入项目通用操作日志。
