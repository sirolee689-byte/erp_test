# 02 - 新增派工单基础资料与单号生成

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/dispatch-order/PRD.md`

## What to build

交付派工单新增的主表基础资料闭环：生产管理人员可以新增未审核派工单，填写派工日期、派工类型、生产车间、交货日期、PI 或委外供应商、备注，系统按派工日期生成 `PGyyMMddxx` 派工单号，并保存主表。

本工单只要求主表能保存成功，并为后续明细选货和整单保存预留接口结构。为了保持业务闭环，可以允许先保存一张“带最小占位明细校验关闭的开发态单据”或使用后端内部测试入口验证主表，但正式页面保存按钮在没有明细时应提示后续工单会接入明细保存，不允许产生正式空单。

本工单不做批量选货、明细落库、超额派工校验、审核、删除。

## Acceptance criteria

- [ ] 新增表单包含派工日期 `scaj02`、派工类型 `scaj03`、生产车间 `scaj05`、交货日期 `scaj06`、关联 PI 或委外供应商、备注 `remark`。
- [ ] 生产车间只能从 `UB_ERP_Stocks_workshop` 选择，保存时写入车间 code 到 `scaj05`，并写入车间名称快照到 `cj`。
- [ ] 派工单号 `scaj01` 按用户填写的派工日期生成，格式为 `PG + yyMMdd + 当日流水号`。
- [ ] 流水号计算包含同日期已删除记录，已删除单据仍占用序号。
- [ ] 保存前做派工单号重复检测；如发生并发重复，后端应重新取号或明确返回可理解错误。
- [ ] 新增主表写入 `pass=0`、`del=0`、`sign=101`、`closed=0`。
- [ ] `systemcode` 按 PRD 规则生成，保证同单唯一。
- [ ] 本厂和大板派工的 `scaj04` 保存关联 PI；委外派工的 `scaj04` 和 `kid` 保存供应商 code。
- [ ] 保存动作写入项目通用操作日志。
