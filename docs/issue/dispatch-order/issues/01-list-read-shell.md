# 01 - 派工单列表与只读详情

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/dispatch-order/PRD.md`

## What to build

交付派工单模块的第一条可演示路径：生产管理人员能从菜单进入派工单页面，看到真实派工单列表，按项目通用规则默认查看已审核、未删除单据，并能打开任意单据查看主表和明细的只读详情。

本工单要打通前后端最小闭环：菜单或路由、权限动作、列表分页接口、单笔详情接口、前端列表页、只读详情页。详情页展示内容接近新增页，但全部只读；第一版不做打印。

本工单不做新增、编辑、选货、审核、反审核、删除、回收站。

## Acceptance criteria

- [ ] 派工单页面不再是占位页，页面位置、标题、操作区风格与项目供应链日常业务模块一致。
- [ ] 列表默认只显示 `del=0` 且 `pass=1` 的派工单，支持分页。
- [ ] 列表提供“显示未审核”开关，开启后显示 `del=0` 且 `pass=0` 的派工单。
- [ ] 搜索条件至少支持派工单号 `scaj01`、PI 或供应商值 `scaj04`、生产车间 `scaj05`、派工类型 `scaj03`、派工日期 `scaj02` 范围、交货日期 `scaj06` 范围。
- [ ] 列表列至少显示派工单号、派工类型、PI 或供应商、生产车间名称、派工日期、交货日期、审核状态、创建人、备注。
- [ ] 单笔详情接口返回主表一行和明细数组，明细包含货品编码、货品名称、规格、单位、版本、本次派工数量、已派工快照、返修数量、PI。
- [ ] 查看详情时所有字段只读，已审核、未审核、已删除状态都不能在本页面编辑。
- [ ] 无数据、接口失败、无权限时有清晰提示，不出现空白页。
