# 09 - 软删除、回收站、恢复与硬删除

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/buy_order/PRD.md`

## What to build

交付采购单删除闭环：未审核、未被入库引用的采购单可以软删除进入回收站；回收站可以恢复误删单据；超级管理员可在严格条件下做硬删除。

本工单沿用新系统回收站模式，不照搬旧系统独立恢复页面。

## Acceptance criteria

- [ ] 有 `delete` 权限的用户可以软删除 `del=0`、`pass=0` 的采购单。
- [ ] 已审核 `pass=1` 的采购单禁止软删除。
- [ ] 存在非删除入库单 `UB_ERP_Stocks_Storage.kcan04 = kcaj01` 时禁止软删除。
- [ ] 软删除成功后主表 `del` 更新为 `1`，并记录删除人和删除时间。
- [ ] 回收站视图只展示 `del=1` 的采购单。
- [ ] 回收站支持按采购单号、供应商、日期等基础条件筛选。
- [ ] 有 `delete` 权限的用户可以恢复 `del=1` 且未审核的采购单。
- [ ] 恢复成功后主表 `del` 回到 `0`，并返回正常列表。
- [ ] 硬删除仅超级管理员可见可用。
- [ ] 硬删除只允许未审核、未被入库引用的采购单。
- [ ] 软删除、恢复、硬删除都写入操作日志。
- [ ] 已删除采购单号仍占用编号，不因恢复或硬删除影响后续编号规则。
