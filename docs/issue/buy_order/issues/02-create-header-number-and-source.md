# 02 - 新增采购单基础资料、单号与关联 PI

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/buy_order/PRD.md`

## What to build

交付采购单新增基础资料闭环：采购员可以创建未审核采购单，选择单号类型、采购类型、供应商、币别、含税方式、关联单号和备注，保存后生成主表记录。

本工单只要求主表基础资料能独立保存成功，并为后续明细、费用和 BOM 快照保存预留同单保存结构。正式页面可以要求后续工单接入明细后再开放完整整单提交，但后端基础资料规则必须先可测试。

## Acceptance criteria

- [ ] 新增表单包含基础资料 Tab，至少支持采购日期 `kcaj02`、采购类型 `kcaj03`、关联单号 `kcaj04`、供应商 `kcaj05`、含税方式 `kcaj06`、币别 `kcaj07`、装货港、卸货港、付款条件、备注。
- [ ] 单号类型支持 `ZY`、`PO`、当前年份三种入口。
- [ ] 选择 `ZY` 时默认 `kcaj03=1` 订单采购，并允许切换到 `kcaj03=2` 请购采购。
- [ ] 选择 `PO` 时默认且锁定 `kcaj03=2` 请购采购，其他采购类型不可选。
- [ ] 选择当前年份时按钮展示当前年份，默认 `kcaj03=2` 请购采购，并允许切换到其他两种采购类型。
- [ ] 采购单号 `kcaj01` 按所选类型生成，并且已删除记录也占用序号，不允许复用旧号。
- [ ] 保存时后端再次检测单号冲突；如被抢占，自动递增到可用单号或返回清晰错误。
- [ ] `kcaj03=1` 订单采购必须填写 `kcaj04`，可手工输入，也可选择单个 PI。
- [ ] `kcaj03=2` 请购采购必须填写 `kcaj04`，可手工输入，也可选择多个 PI，多个 PI 保存为逗号分隔文本。
- [ ] `kcaj03=0` 其他采购允许 `kcaj04` 为空。
- [ ] 供应商只能选择已审核、未删除、类别为采购或共用的 `UB_ERP_System_supplier`。
- [ ] 保存时后端按 `kcaj05` 重新查询供应商名称并写入 `kehu`，不信任前端传入名称。
- [ ] 币别只能选择已审核、未删除的 `UB_ERP_Finance_currency`。
- [ ] 保存时后端按 `kcaj07` 重新查询币别名称和汇率，写入 `rmb` 与 `rmb_hl`。
- [ ] 新增保存后主表写入 `pass=0`、`del=0`、`closed=0`，并写入创建人、创建时间和操作日志。
