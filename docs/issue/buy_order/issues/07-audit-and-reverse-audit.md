# 07 - 单条审核与反审核

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/buy_order/PRD.md`

## What to build

交付采购单审核闭环：审核员可以对未审核采购单做单条审核，审核通过后采购单成为后续采购入库的有效来源；已审核采购单可以反审核，但必须填写原因，且已有有效入库关联时禁止反审核。

第一版不做批量审核，不做独立驳回状态，也不发送旧系统的超 30000 RMB 邮件。

## Acceptance criteria

- [ ] 有 `audit` 权限的用户可以审核 `del=0`、`pass=0` 的采购单。
- [ ] 审核前后端校验采购单至少有一条有效采购明细。
- [ ] 审核成功后主表 `pass` 更新为 `1`，并记录审核人、审核时间或现有审核字段。
- [ ] 审核成功后采购单在列表状态变为已审核。
- [ ] 已审核 `pass=1` 的采购单前端隐藏或禁用直接编辑和删除入口。
- [ ] 有 `audit` 权限的用户可以对 `pass=1` 的采购单发起反审核。
- [ ] 反审核必须填写原因，原因为空时拒绝。
- [ ] 反审核原因写入 `UB_ERP_Buy_order_sp`。
- [ ] 如存在非删除入库单 `UB_ERP_Stocks_Storage.kcan04 = kcaj01`，反审核必须被拒绝。
- [ ] 反审核成功后主表 `pass` 回到 `0`。
- [ ] 审核和反审核都写入操作日志，反审核日志包含原因摘要。
- [ ] 第一版不提供批量审核和审核驳回状态。
