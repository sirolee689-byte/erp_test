# 01 - 采购单列表、筛选与页面壳

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/buy_order/PRD.md`

## What to build

交付采购单模块的第一条可用路径：用户进入采购单菜单后看到真实列表，而不是占位页。列表默认展示 `del=0` 的采购单，包含未审核、已审核和已结案单据，并支持按审核状态、结案状态、采购类型、供应商、日期范围和关键词筛选。

本工单建立后续新增、编辑、审核、删除、打印等动作入口，但未完成的动作可以先置灰或提示“后续接入”。重点是把页面壳、读接口、权限入口、状态显示和列表搜索口径先打通。

## Acceptance criteria

- [ ] 采购单菜单打开后展示采购单列表，不再显示占位内容。
- [ ] 默认列表只展示 `UB_ERP_Buy_order.del=0` 的数据，已结案 `closed=1` 仍在正常列表中展示。
- [ ] 支持按 `pass`、`closed`、`kcaj03`、`kcaj05`、采购日期范围和关键词筛选。
- [ ] 关键词至少能匹配采购单号 `kcaj01`、关联单号 `kcaj04`、供应商快照 `kehu` 和备注。
- [ ] 列表展示采购单号、日期、采购类型、关联单号、供应商、币别、审核状态、结案状态、创建人、创建时间等核心字段。
- [ ] 列表行操作按状态预留查看、编辑、审核、反审核、结案、反结案、删除、打印入口。
- [ ] 无对应权限时隐藏或禁用对应按钮，至少覆盖 `view/add/edit/audit/close/delete/price/export`。
- [ ] 无 `price` 权限时列表不展示价格、税点、金额、合计等敏感字段。
- [ ] 读接口使用旧系统采购单真实字段：采购单号 `kcaj01`、供应商编码 `kcaj05`、供应商名称 `kehu`。
- [ ] 列表读取不使用当前未完善代码中的 `cgad01` / `cgad05` 作为采购单字段。
