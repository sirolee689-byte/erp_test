# 12 - 采购单端到端验收与加固

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/buy_order/PRD.md`

## What to build

交付采购单第一版最终验收：串起新增、编辑、审核、打印、采购入库引用、结案、回收站和权限场景，补齐遗漏测试和文档说明，确保采购单模块可以作为后续采购入库的正式来源。

本工单不新增 PRD 之外的大功能，主要做端到端验证、边界修复和文档同步。

## Acceptance criteria

- [ ] 完成一条完整路径：新增采购单、选择供应商/币别/PI、添加明细和费用、保存、查看、审核、打印。
- [ ] 完成采购入库引用路径：已审核未结案采购单可被采购入库选择，入库保存后采购单禁止反审核和删除。
- [ ] 完成结案路径：有有效入库引用的已审核采购单可结案，结案后采购入库不可再选择。
- [ ] 完成回收站路径：未审核未引用采购单可软删除、恢复，并保持单号占用规则。
- [ ] 完成价格权限路径：无 `price` 权限用户在列表、详情、编辑、打印中都看不到也改不了价格金额。
- [ ] 完成操作日志验收：关键动作都有日志记录，能按采购单号追溯。
- [ ] 补齐或更新采购单模块开发说明，明确 `kcaj01/kcaj05/kehu/kcak01/kcak02/kcak03` 等关键字段口径。
- [ ] 运行相关自动化测试，至少覆盖 PRD Testing Decisions 中列出的核心场景。
- [ ] SQL 写法保持 SQL Server 2008 R2 兼容，不使用 `TRY_CONVERT`、`OFFSET`、`FORMAT`、`IIF`、`CONCAT` 等不兼容语法。
- [ ] 如本轮改动涉及 `server/**`，按项目规则重启 API 并确认新进程启动成功。
- [ ] 明确记录第一版未做事项：批量审核、驳回状态、真实 Excel 导出、超 30000 RMB 邮件、超订量申请复核。
