# 08 - 采购单结案与反结案

**Status:** `ready-for-agent`  
**Type:** AFK

## Parent

`docs/issue/buy_order/PRD.md`

## What to build

交付采购单结案闭环：管理员可以把已有入库记录的已审核采购单结案，结案后不再允许后续采购入库引用；超级管理员可以作为异常修复执行反结案。

结案不要求采购数量全部入库，只要求至少存在一张有效入库单作为业务依据。

## Acceptance criteria

- [ ] 有 `close` 权限的用户可以对 `del=0`、`pass=1`、`closed=0` 的采购单发起结案。
- [ ] 结案前后端校验存在至少一张非删除入库单 `UB_ERP_Stocks_Storage.kcan04 = kcaj01`。
- [ ] 没有有效入库关联时结案被拒绝，并给出明确提示。
- [ ] 结案不校验是否全部采购数量已入库。
- [ ] 结案成功后主表 `closed` 更新为 `1`。
- [ ] `closed=1` 的采购单在正常列表仍可见，但状态显示为已结案。
- [ ] `closed=1` 的采购单不能再被采购入库来源选择。
- [ ] 反结案仅作为超级管理员异常修复动作开放。
- [ ] 反结案成功后主表 `closed` 回到 `0`。
- [ ] 结案和反结案写入操作日志。
