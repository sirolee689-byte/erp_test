# 09 - 端到端验收与边界加固

**Status:** `ready-for-agent`  
**Type:** AFK  

## Parent

`docs/issue/assist_order/PRD.md`

## What to build

对外协订单第一版做完整验收收口：从新增、保存、审核、结案、查询、打印到回收站清理走完一遍真实业务流程，并补齐前面工单没有覆盖到的边界提示、空数据状态、错误文案和回归测试。

本工单不新增 PRD 外的新功能，只做验收、补洞和文档化测试证据。

## Acceptance criteria

- [ ] 用户可以完整完成一张外协订单从新增、保存、审核、结案到打印的流程。
- [ ] 用户可以反结案、反审、修改、再次保存、再次审核。
- [ ] 用户可以软删除未审核未结案单据，在回收站恢复，也可以彻底删除回收站中的未审核单据。
- [ ] 已审核、已结案、回收站等状态下的禁止动作都有清晰提示。
- [ ] 订单外协/订单外发缺少关联单号时保存失败；其他外协允许关联单号为空。
- [ ] 明细和额外费用整批重写后刷新不丢数据。
- [ ] 列表默认、未审核、回收站、结案筛选、供应商筛选、外协类型筛选、排序都经过验收。
- [ ] 打印包含表头、明细、额外费用、合计、合同条款、签字栏，并能批量连续打印。
- [ ] 所有关键动作都能在 `UB_Date_ERP_Operation_log` 查到记录。
- [ ] 完成一份简短验收记录，写明测试账号、测试数据、跑过的命令或手工路径、剩余风险。
