# 02 - 新增和编辑外协订单基础资料

**Status:** `ready-for-agent`  
**Type:** AFK  

## Parent

`docs/issue/assist_order/PRD.md`

## What to build

交付外协订单的基础资料录入闭环：用户可以新增未审核单据，编辑未审核未结案单据，并保存主表基础字段。保存时由后端负责最终单号唯一性、供应商名称快照和币别名称快照。

页面使用当前 ERP 页面内的大弹窗或抽屉，不做浏览器原生新页。新增时系统建议外协订单号，用户可改；最终保存时如果单号已存在，后端自动递增到可用单号并把最终单号返回给前端提示。

本工单只保存主表基础资料，可以暂时不要求明细和额外费用完整录入；但保存接口要为后续主表、明细、费用同单事务保存预留结构。

## Acceptance criteria

- [ ] 新增时可录入 `wxaj02`、`wxaj03`、`wxaj04`、`wxaj05`、`wxaj06`、`wxaj07`、`wxaj08`、`remark`、`notes`、单价小数位等基础字段。
- [ ] 外协订单号建议规则为 `WX` + 保存日期 `yyMMdd` + 流水号，例如 2026-06-09 第一张为 `WX26060901`。
- [ ] 流水号 `01` 到 `99` 保持两位，超过后自然变成三位，例如 `WX260609100`。
- [ ] 保存时如果当前 `wxaj01` 已存在，系统自动递增到下一个可用号，保存成功后提示最终保存的单号。
- [ ] `wxaj03=0` 时 `wxaj04` 可空；`wxaj03=1` 或 `wxaj03=2` 时 `wxaj04` 必填；第一版不校验关联单号是否真实存在。
- [ ] 外协商下拉只选择 `UB_ERP_System_supplier` 中 `s_lb` 为外协或共用、已审核、未删除的数据。
- [ ] 保存时后端根据 `wxaj05` 实时查询供应商名称并覆盖写入 `kehu`，不信任前端传入名称。
- [ ] 币别下拉只选择 `UB_ERP_Finance_currency` 中已审核、未删除的数据。
- [ ] 保存时后端根据 `wxaj07` 实时查询币别名称并覆盖写入名称快照字段。
- [ ] `wxaj06` 只保存 `1` 含税或 `2` 不含税，保存逻辑不根据该字段重算金额。
- [ ] 单价小数位默认 `4`，输入范围限制为 `0` 到 `6`，沿用主表现有 `decimal` 字段。
- [ ] 已审核或已结案单据禁止保存修改，前端按钮置灰，后端也拒绝。
