# 05 - 审核、结案、删除与操作日志

**Status:** `ready-for-agent`  
**Type:** AFK  

## Parent

`docs/issue/assist_order/PRD.md`

## What to build

交付外协订单生命周期闭环：用户可以对单据执行审核、反审、结案、反结案、软删除、恢复、彻底删除。每个动作都要遵守状态限制，并写入 `UB_Date_ERP_Operation_log`。

第一版只做审核和反审，不做审核不通过。已结案单据不能直接反审，必须先反结案，再反审。

## Acceptance criteria

- [ ] 未审核单据可编辑、可软删除、不可结案。
- [ ] 已审核未结案单据只读、不可删除、可结案。
- [ ] 已审核已结案单据只读、不可删除、可反结案。
- [ ] 回收站单据不可结案、不可反结案。
- [ ] 审核后主表、明细、额外费用全部只读；反审后未结案单据可重新编辑。
- [ ] 已结案单据直接反审会被拒绝，并提示必须先反结案。
- [ ] 已审核单据删除前必须先反审；已结案单据删除前必须先反结案再反审。
- [ ] 软删除只允许未审核未结案单据，写主表删除标记，不物理删除明细和费用。
- [ ] 恢复允许从回收站恢复。
- [ ] 彻底删除只允许回收站中的未审核单据，并同时物理删除明细和额外费用。
- [ ] 列表和详情里的按钮根据状态正确显示、禁用或隐藏。
- [ ] 新增、修改、删除、恢复、审核、反审、结案、反结案都写入 `UB_Date_ERP_Operation_log`。
- [ ] 日志 `act_name` 沿用 PRD：外协订单录入、外协订单修改、外协订单删除、被删除外协订单恢复成功、申请审核、申请反审核、申请结案、申请反结案。
- [ ] 第一版没有“审核不通过”按钮、接口和日志。
